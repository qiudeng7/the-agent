# Electron + Vite + Vue 模板使用指南

## 快速开始

### 1. 安装依赖

```bash
pnpm install
```

**背后发生了什么：**
- pnpm 从 `package.json` 读取依赖列表
- 安装 Vue 3、Vite、Electron、Pinia、Vue Router 等核心库
- 安装 Electron Forge 及其 makers（用于打包）
- 生成 `pnpm-lock.yaml` 锁定版本
- 使用硬链接和内容寻址存储节省磁盘空间

---

### 2. 开发模式

```bash
pnpm run dev
```

**背后发生了什么：**

1. **Vite 启动开发服务器**
   - Vite 监听 `src/` 目录下的文件变化
   - Vue 插件编译 `.vue` 单文件组件
   - 提供 HMR（热模块替换），修改代码后浏览器/Electron 自动更新

2. **vite-plugin-electron 构建主进程**
   - 监听 `electron/main/index.ts`
   - 编译 TypeScript 为 JavaScript
   - 输出到 `dist-electron/main/`

3. **vite-plugin-electron-renderer 构建预加载脚本**
   - 监听 `electron/preload/index.ts`
   - 编译后输出到 `dist-electron/preload/`
   - 文件变化时自动重启 Electron

4. **Electron 启动**
   - 加载开发服务器 URL（通常是 `http://localhost:5173`）
   - 创建浏览器窗口显示 Vue 应用

**目录结构：**
```
├── electron/
│   ├── main/          # 主进程代码（Node.js 环境）
│   └── preload/       # 预加载脚本（隔离环境）
├── src/
│   ├── views/         # 页面组件
│   ├── stores/        # Pinia 状态管理
│   ├── router/        # 路由配置
│   └── main.ts        # Vue 应用入口
```

---

### 3. 构建应用

```bash
pnpm run build    # 构建并打包
```

**背后发生了什么：**

1. **TypeScript 类型检查** (`vue-tsc`)
   - 检查所有 `.ts` 和 `.vue` 文件的类型
   - 发现类型错误时构建失败

2. **Vite 生产构建**
   - 打包所有 Vue 组件、CSS、静态资源
   - Tree-shaking 移除未使用代码
   - 代码压缩和混淆
   - 输出到 `dist/` 目录

3. **Electron Forge 打包**
   - `@electron-forge/maker-dmg` → macOS 安装包
   - `@electron-forge/maker-zip` → macOS 压缩包
   - `@electron-forge/maker-squirrel` → Windows 安装程序
   - `@electron-forge/maker-rpm` → Linux RPM 包
   - `@electron-forge/maker-deb` → Linux DEB 包
   - 输出到 `out/` 目录

**输出示例：**
```
out/
├── The Agent-darwin-arm64/
│   └── The Agent.app
├── The Agent-darwin-arm64.dmg
├── The Agent-win32-x64/
│   └── The Agent.exe
└── ...
```

## 核心概念

### 主进程 vs 渲染进程

| 主进程 (Main) | 渲染进程 (Renderer) |
|--------------|-------------------|
| `electron/main/index.ts` | `src/` 下的 Vue 组件 |
| Node.js 完整 API | 受限的浏览器环境 |
| 创建窗口、菜单、托盘 | 显示 UI、响应用户交互 |
| IPC 通信的服务端 | IPC 通信的客户端 |

### IPC 通信流程

```
渲染进程 (Vue 组件)
      ↓
window.electronAPI.xxx()
      ↓
contextBridge (preload/index.ts)
      ↓
ipcRenderer.invoke('channel')
      ↓
     IPC
      ↓
ipcMain.handle('channel', handler)
      ↓
electron/main/index.ts (执行系统级操作)
```

**示例：打开文件对话框**

1. **预加载脚本暴露 API** (`electron/preload/index.ts`)
```typescript
contextBridge.exposeInMainWorld('electronAPI', {
  openFileDialog: () => ipcRenderer.invoke('open-file-dialog'),
})
```

2. **主进程处理请求** (`electron/main/index.ts`)
```typescript
ipcMain.handle('open-file-dialog', async () => {
  const { dialog } = await import('electron')
  return dialog.showOpenDialog(mainWindow, { ... })
})
```

3. **Vue 组件调用** (`src/views/Settings.vue`)
```typescript
const result = await window.electronAPI.openFileDialog()
```

### 状态管理 (Pinia)

```typescript
// src/stores/agent.ts
import { defineStore } from 'pinia'

export const useAgentStore = defineStore('agent', () => {
  const tasks = ref([])
  const addTask = (name) => { ... }
  return { tasks, addTask }
})
```

**使用方式：**
```vue
<script setup>
import { useAgentStore } from '@/stores/agent'
const store = useAgentStore()
store.addTask('新任务')
</script>
```

### 路由 (Vue Router)

```typescript
// src/router/index.ts

  history: createWebHashHistory(), // Electron 推荐用 Hash 模式
  routes: [
    { path: '/', component: Home },
    { path: '/settings', component: Settings },
  ],
})
```

**导航：**
```vue
<router-link to="/settings">设置</router-link>
```

## 扩展开发

### 添加新页面

1. 创建组件 `src/views/About.vue`
2. 添加路由 `src/router/index.ts`
3. 导航链接添加菜单项

### 添加新的 IPC 通道

1. `electron/preload/index.ts` - 暴露新方法
2. `electron/main/index.ts` - 添加 `ipcMain.handle`
3. `src/electron.d.ts` - 更新类型定义
4. Vue 组件中调用

### 添加系统托盘

在主进程中：
```typescript
import { Tray, Menu } from 'electron'

let tray: Tray | null = null
tray = new Tray('path/to/icon.png')
tray.setContextMenu(Menu.buildFromTemplate([
  { label: '退出', click: () => app.quit() }
]))
```

## 常见问题

### Q: 为什么 DevTools 自动打开？
A: 开发模式下默认开启，可在 `electron/main/index.ts` 中注释掉 `openDevTools()`

### Q: 如何调试主进程？
A: 在 VS Code 中附加到 Electron 进程，或使用 `console.log` 输出到终端

### Q: 打包后白屏？
A: 检查 `vite.config.ts` 中的 `base: './'` 确保资源路径正确

### Q: 如何添加原生 Node 模块？
A: 运行 `pnpm add <module>` 后，Electron Forge 会自动在 `rebuildConfig` 阶段重新编译原生模块
