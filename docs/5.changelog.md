# Changelog

All notable changes to this project will be documented in this file.

## [0.9.1] - 2026-03-29

### Changed

- 重构 Chat.vue 职责拆分，代码从 984 行减少到 246 行：
  - 提取 ThinkingBlock.vue 组件：思考块展示，支持折叠/展开
  - 提取 ToolBlock.vue 组件：工具调用块展示，支持 Tab 切换
  - 提取 MessageItem.vue 组件：消息项展示，封装 UI 状态管理
  - 创建 useChatSubmit composable：封装消息提交、模型初始化逻辑
- 移除 ChatInput.vue 中的权限模式持久化逻辑，改由 useChatSubmit 统一处理

### Fixed

- 添加窗口关闭时的事件监听器清理，避免内存泄漏
- 修复 ui-settings.ts 持久化职责边界，由 settings.ts 统一处理
- 补充 ClaudeProviderOptions 接口注释，移除未实现的 getVersion 方法

## [0.9.0] - 2026-03-29

### Changed

- 重构 claude provider 模块，移除过度设计：
  - 删除 IClaudeProvider 接口（只有一个实现，无需抽象层）
  - 删除 content-mapper.ts（映射逻辑内联到 provider）
  - 使用 SDK 导出的类型（SDKMessage、BetaContentBlock 等）
  - 代码减少约 60 行，更易维护

### Fixed

- 修复工具展开状态共享问题：每个 tool 组件现在有独立的展开状态
- 修复 tool_result 显示：正确从 SDK user 消息中提取并显示工具结果

## [0.8.0] - 2026-03-29

### Added

- 权限模式选择器：支持 default、auto、acceptEdits、bypassPermissions、plan 五种模式
- 权限模式用户设置：自动保存到云端，应用启动时自动恢复
- 停止生成按钮：生成过程中可点击停止，显示红色方形图标
- 工具执行状态显示：显示"执行中..."、"已完成"或"失败"
- 工具详情 Tab 切换：展开后可切换查看"参数"和"结果"

### Changed

- 重构对话框底部工具栏：
  - 模型选择器移到工具栏左侧
  - 权限模式选择器放在模型选择器旁边
  - 移除深度思考、联网搜索、工具按钮
- 流式消息按实际顺序显示：thinking、tool_use、text 按照生成顺序排列

### Fixed

- 修复流式状态跨 session 显示：切换 session 时只显示当前 session 的生成状态
- 修复 tool_use 块在流式生成中不显示：实时显示 Bash、Glob 等工具调用
- 修复输入法组合输入时按回车误发送消息的问题

### Fixed

- 修复流式输出索引问题：多消息时正确映射 SDK 块索引
- 修复工具调用显示：添加 message_start/message_stop 事件处理

### Changed

- 新消息自动滚动到底部
- 工具调用支持展开查看详细信息
- 工具结果支持展开查看

## [0.7.0] - 2026-03-29

### Added

- Agent 流式输出支持：启用 SDK `includePartialMessages` 选项，AI 回复实时逐字显示
- 流式事件类型：text_delta、thinking_delta、input_json_delta、content_block_start、content_block_stop

### Changed

- 优化流式增量事件处理，支持文本和思考内容的实时累积

## [0.6.0] - 2026-03-29

### Added

- Claude Agent SDK 集成：使用 @anthropic-ai/claude-agent-sdk 替代原有 agent 模块
- claude-installer 模块：自动检测系统 Claude Code 安装
- Nitro 后端服务：支持用户认证、会话管理、消息存储

### Changed

- 重构 agent 模块：删除通用抽象层，全面使用 claude 模块
- 优化状态管理：依赖倒置、模块拆分、事件生命周期管理

### Fixed

- 修复 AI 回复不显示在前端的问题（Vue 响应式追踪缺陷）
- 修复流式内容 computed 属性缺失问题
- 修复用户消息重复发送问题
- 修复事件监听未清理导致的内存泄漏
- 修复 mediaQuery 监听泄漏
- 修复 Store 封装违规（直接操作子模块内部状态）
- 修复路由守卫未处理 auth init 错误
- 修复 IPC handler 重复注册风险
- 修复设置同步失败无反馈

## [0.5.0] - 2026-03-28

### Added

- 思考内容折叠功能：AI 消息中的思考过程（thinking block）支持折叠/展开
- 折叠状态记忆：默认折叠，用户展开后记住状态

### Fixed

- 修复思考折叠 toggle 在默认展开时的首次点击失效问题

## [0.4.0] - 2026-03-28

### Added

- Markdown 实时渲染：AI 消息支持实时渲染 Markdown 格式（代码块、列表、标题、链接等）
- MarkdownRenderer 组件：基于 marked 库的 GFM 格式渲染组件

## [0.3.0] - 2026-03-26

### Added

- 会话模型状态：每个会话记住当前使用的模型，切换会话时自动恢复
- 消息模型记录：assistant 消息记录使用的模型 ID
- 内置模型：预置 Qwen3.5-Plus、GLM-5、Kimi-K2.5、MiniMax-M2.5 四个内置模型

### Changed

- 重构模型管理：分离内置模型（bundle）和自定义模型（custom）
- 首页创建会话时继承当前选择的模型

## [0.2.1] - 2026-03-26

### Changed

- 移除默认模型列表，用户首次使用需在设置中自行添加模型
- 无模型时显示提示引导用户前往设置页面

## [0.2.0] - 2026-03-25

### Added

- 自定义模型设置：支持添加、删除、选择默认模型
- API Base URL 设置：支持代理和自托管服务
- 流式消息展示：实时显示 AI 思考过程和回复内容
- Electron IPC Transport：主进程与渲染进程通信机制
- ClaudeProvider：基于 @anthropic-ai/sdk 的流式 agentic loop
- AgentRunner：工具执行机制

### Fixed

- 修复 API Key 和 baseURL 设置不生效问题
- 修复首页和聊天页模型切换无效问题
- 修复默认模型设置无效问题
- 修复 reactive proxy 通过 IPC 传输失败问题
- 修复首页输入框消息发送和回车键行为
- agentStore 错误处理，追加错误消息到聊天记录
